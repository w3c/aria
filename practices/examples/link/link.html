Link design pattern

The example on this page is based on the Link design pattern in the ARIA 1.1 Authoring Practices Guide.

Note: When using HTML use the <a> element. Do not re-purpose other elements to create links.

Example

W3C website

Code

HTML


<span tabindex="0" role="link" id="link">W3C website</span>

JavaScript


function init()
{
        document.getElementById("link").addEventListener("click", fetchResource);
        document.getElementById("link").addEventListener("keydown", fetchResource);
}

function fetchResource(event)
{
        var type = event.type;
        
        if (type === "keydown" && event.keyCode !== 13)
        {
                return true
        }

        window.location.href = 'http://w3.org/';
        event.preventDefault();
}

window.onload = init;

Notes

Uses the ARIA role attribute with a value of link. This informs the browser that it should expose the link as a link through its accessibility API.

Uses the HTML tabindex attribute with a value of 0. This places the link into the tab order based on its location in the source order. The visual appearance of the link also changes when focus moves to it.

Keyboard interaction for the link is provided through scripting. The enter key event is captured, and used to trigger the link's expected behaviour.
